# Runtime state

`ConstraintState` is the live configuration after a generated byte prefix.

It owns:

- the active tokenizer-state/frontier map;
- Commit scratch buffers;
- the Mask cache and Mask scratch buffers;
- a generation counter used only for cache invalidation.

It does not own the compiled grammar, tokenizer, Parser DWA, or token masks.
Those live in `Constraint` and are borrowed immutably.

Files:

- `mod.rs`: the `ConstraintState` struct, clone/debug semantics, and module exports.
- `cache.rs`: cached materialized Mask output and Mask dense scratch.
- `scratch.rs`: reusable Commit buffers.
- `inspect.rs`: read-only parser/frontier observations.
- `force.rs`: derived forced-token convenience operation implemented using Mask/Commit.
