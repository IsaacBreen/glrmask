# Paper terminology mapping

The paper-level operation is `Commit`. This source chunk maps it as follows:

- generated byte/token prefix: input to `api.rs`;
- lexer scan relation: implemented by `tokenizer_scan.rs` plus `initial_scan.rs`;
- completed terminal sequence: represented by normalized matches in `acceptance.rs`;
- parser stack-effect recognizer: invoked through `parser_advance.rs` and specialized in `single_top.rs`;
- delayed longest-match exclusions: maintained in `pruning.rs`;
- active runtime state: updated by `general.rs`, `fast_path.rs`, and `queue.rs`;
- diagnostics: isolated in `profiled.rs` and `profile.rs`.

This is deliberately not LR-specific in naming. The parser machinery may currently be GLR internally, but Commit consumes stack-effect semantics through the parser-advance boundary.
